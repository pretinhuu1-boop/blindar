# Runbook — Conscientização de usuário

> ⚠ **Fora de escopo de código.** Template de programa de treinamento.

## Por que isso existe

Frameworks (ISO 27001 A.6, NIST CSF PR.AT, CIS Control 14) exigem
treinamento periódico de usuários. O skill não treina ninguém — só lista
o mínimo viável pra fechar o gap.

## Programa mínimo (anual)

| Quando | Conteúdo | Audiência |
|---|---|---|
| Onboarding | NDA + política de seg + golpes comuns (phishing, vishing) | todos novos |
| Trimestral | Phishing simulado (campanha real, sem aviso) | todos |
| Semestral | Atualização de ameaças relevantes ao setor | todos |
| Anual | Treinamento formal com tracking de conclusão | todos |
| Por função | Devs: secure coding. Sysadmins: hardening. Comercial: golpes de fatura. | função |

## Indicadores

| Métrica | Alvo |
|---|---|
| Taxa de clique em phishing simulado | < 5% após 12 meses de programa |
| Taxa de reportar phishing (canal interno) | > 30% |
| Conclusão treinamento anual | 100% |
| Tempo médio de reporte de e-mail suspeito | < 1 dia |

## Ferramentas comuns

- KnowBe4
- Hoxhunt
- Curso interno + Google Forms (low-budget)
- Phishing simulado: GoPhish (open-source)

## Por que NÃO está num agente

O skill não tem como executar treinamento — depende do RH/L&D + gerentes.
Listamos pra fechar o gap formal de auditoria.
